## 开发规范

### 语言偏好
- 所有代码注释、文档说明、错误提示等均使用中文
- 必要时可保留英文专业术语以确保准确性
- 思考过程中的选项也必须使用中文输出

### 任务执行规范

#### 任务开始
- 任务开始时，必须先说"开始"，再执行任务

#### 任务进行中
- 任务执行过程中，需要实时显示进度
- 使用进度条显示百分比（如：`[████████░░] 80%`）
- 进度更新频率：每完成一个关键步骤或每 10-20% 更新一次

#### 任务中断
- 任务执行中途需要进行选择时，提醒用户"请进行选择"
- 任务执行中途需要中断时，提醒用户"任务已中断"

#### 任务完成
- 任务完成后，必须说"完成"，进行结束
- 如果不在主界面，需要提醒用户返回主界面

### 进度显示格式
```
开始 [任务描述]

进度: [████████░░░░░░░░░░░░] 20% - [当前步骤]
进度: [████████████████░░░░] 80% - [当前步骤]

完成！[任务结果总结]
```

## 常见任务

### 创建新项目
```bash
# Node.js 项目
npm init -y

# Python 项目
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或
venv\Scripts\activate  # Windows
```

### 运行测试
```bash
# 根据项目类型选择相应的测试命令
npm test  # Node.js
pytest    # Python
```

### 代码检查
```bash
# 根据项目类型选择相应的检查命令

```
